## Search Results

**For X (Twitter) API v2 in 2026, use **Bearer Token** (OAuth 2.0 App-Only) for read-only access to public data, and **OAuth 2.0 Authorization Code Flow with PKCE** for user-context actions like posting or accessing private data.** Bearer Token is simpler and sufficient for most app-only needs, while full OAuth 2.0 is required for user actions and recommended for new apps due to enhanced security[1][3][5].

### Key Differences: Bearer Token vs. OAuth 2.0
| Aspect | Bearer Token (App-Only) | OAuth 2.0 (User Context) |
|--------|--------------------------|---------------------------|
| **Use Case** | Read public data (e.g., timelines, searches). No user permission needed[2][3]. | Write actions (post, like) or private data. Requires user authorization[3][5]. |
| **Security** | App authenticates itself; keep token secret. Simpler but limited scope[1][2]. | Modern PKCE flow; granular scopes. Preferred for new apps[3][7]. |
| **Setup Complexity** | Generate once from developer portal[1][6]. | Enable in app settings, implement 3-legged flow or code flow[3][4]. |
| **Node.js/TS Fit** | Ideal for server-side scripts; use `fetch` or `axios` with header[1]. | Use libraries like `oauth4webapi` for secure PKCE[3]. |

**Never use Bearer Token for user actionsâ€”it lacks user context and will fail[5].** Store all keys (API Key, Secret, Bearer, Client ID/Secret) in environment variables, never in code or Git[6].

### Step-by-Step Setup in Node.js/TypeScript (2026 Best Practices)
1. **Create App in X Developer Portal**:
   - Log in at developer.x.com, create Project > App.
   - Set permissions: Read+Write+Direct Messages if needed.
   - Go to **Settings > User Authentication Settings** > Enable OAuth 2.0[3][4][6].
   - Generate/regenerate: API Key, API Secret, **Bearer Token**, Client ID/Secret[1][6].

2. **App-Only Bearer Token Example** (for public reads):
   ```typescript
   // Install: npm i node-fetch dotenv
   import fetch from 'node-fetch';
   import * as dotenv from 'dotenv';
   dotenv.config();

   const BEARER_TOKEN = process.env.X_BEARER_TOKEN!;

   async function getPublicTweets(query: string): Promise<void> {
     const response = await fetch(
       `https://api.twitter.com/2/tweets/search/recent?query=${encodeURIComponent(query)}`,
       {
         headers: {
           Authorization: `Bearer ${BEARER_TOKEN}`,
         },
       }
     );
     const data = await response.json();
     console.log(data);
   }

   getPublicTweets('from:elonmusk');  // Example: Recent tweets from @elonmusk[1][2]
   ```
   - **Rate Limits**: Higher in v2; check response headers[1].

3. **OAuth 2.0 User Context Example** (for posting; use PKCE for security):
   ```typescript
   // Install: npm i oauth4webapi axios dotenv
   import { OAuth2Client } from 'oauth4webapi';
   import axios from 'axios';
   import * as dotenv from 'dotenv';
   dotenv.config();

   const client = new OAuth2Client({
     client_id: process.env.X_CLIENT_ID!,
     token_endpoint_auth_method: 'client_secret_post',  // Or 'private_key_jwt' for advanced
   });

   // Step 1: Get authorization URL (PKCE recommended)
   const authUrl = client.code.getAuthorizationUrl({
     scope: 'tweet.read tweet.write users.read offline.access',  // Granular scopes[3]
     redirect_uri: 'http://localhost:3000/callback',
   });
   console.log('Visit:', authUrl.toString());

   // Step 2: After user authorizes, exchange code for tokens (server-side)
   async function exchangeCode(code: string) {
     const tokenSet = await client.code.exchangeAuthorizationCode({
       code,
       redirect_uri: 'http://localhost:3000/callback',
       client_secret: process.env.X_CLIENT_SECRET!,  // Securely stored
     });
     return tokenSet.access_token;
   }

   // Step 3: Use access token to post
   async function postTweet(accessToken: string, text: string): Promise<void> {
     const response = await axios.post(
       'https://api.twitter.com/2/tweets',
       { text },
       { headers: { Authorization: `Bearer ${accessToken}` } }
     );
     console.log('Posted:', response.data);
   }
   ```
   - **PKCE Note**: For SPAs/mobile, generate code challenge/verifier client-side[3].
   - **Refresh Tokens**: Use `offline.access` scope for long-lived access[3].

### Best Practices (2026)
- **Security**: Rotate keys regularly; use HTTPS; validate scopes minimally[3][6][9].
- **Error Handling**: Catch 401/429; implement exponential backoff[1].
- **Libraries**: `@twitter-api-v2/node` or `twitter-api-v2` for full abstraction (check 2026 updates).
- **Quotas**: Free tier limited; upgrade for production[3].
- **Testing**: Use X's API tester or Apidog[4].

If needing enterprise features, consider HTTP Basic or OAuth 1.0a as fallbacks, but OAuth 2.0 is the forward path[2]. Test in sandbox first[8].
