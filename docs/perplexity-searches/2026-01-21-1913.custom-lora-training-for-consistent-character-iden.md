### Common Problems and Step-by-Step Fixes
1. **Poor face consistency (e.g., side profiles drift)**: Add more angled face images/captions. Fix: Retrain with 20% more dataset variety.[4]
2. **Overfitting (memorizes poses, not generalizes)**: Too many steps/images of same pose. Fix: Stop at 3k-5k steps, test on new prompts/samplers (Euler a, 20-30 steps).[2][4]
3. **Inconsistent eyes/body in NSFW**: Weak captions or low-res dataset. Fix: Use BLIP/DeepDanbooru for auto-tags + manual "1girl, nsfw, detailed face" emphasis.[2][4]
4. **Bad generalization across checkpoints**: Train on target base (e.g., Flux.1-dev for NSFW). Fix: Vary samplers/strength in testing (e.g., <lora:char:0.8>).[2]
5. **Setup errors**: Kohya_ss config mismatches (e.g., wrong resolution 1024x1024 for SDXL). Fix: Follow params from HF discussions—network_dim=32, alpha=16 for Flux/SDXL humans.[6]
