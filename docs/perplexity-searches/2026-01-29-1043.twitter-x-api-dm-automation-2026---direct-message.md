### X (Twitter) DM API Limits and Automation in 2026

X enforces a **daily limit of 500 direct messages (DMs) sent per account**, applicable to both manual and API-automated sending. This is a hard account-level cap that resets daily; exceeding it triggers errors until the next day.[1] Automated tools recommend staying far below at **50-80 DMs/day** to avoid restrictions, with built-in pacing to mimic human behavior.[3][4]

#### 24-Hour Window and Time Limit Restrictions
- No strict **24-hour window restriction** like messaging platforms (e.g., WhatsApp's 24-hour session after user initiation). X DMs can be sent proactively to any user who allows messages from your account (e.g., followers or verified users), without requiring prior user response.[1][5]
- **Can you send DMs without time limit?** Yes, beyond the daily 500 cap, there's no per-conversation time window via API. Use endpoints like `POST /2/dm_conversations/:id/messages` for ongoing automation (Basic tier $200/month required; free tier restricted).[5][7]
- **Rate limits**: API v2 uses **15-minute request windows** (e.g., 450 requests/15 mins for search on Basic tier) plus monthly post quotas (500K posts/month Basic). DM endpoints have per-endpoint limits; hitting them returns HTTP 429 with `Retry-After`.[5] Account-level DMs remain 500/day.[1]

#### Webhook Real-Time Events
X API v2 supports **webhooks for real-time DM events** via Account Activity API (legacy v1.1) or filtered streams in v2, enabling instant notifications for incoming DMs/conversations. Integrate for automation like auto-replies, but requires Pro tier or higher for full streaming; Basic allows DM retrieval (`GET /2/dm_events`).[5][7]

#### Bot Detection and Safety for Automation
- **Bot detection**: X flags aggressive patterns (e.g., identical messages, rapid sending, links in first DM). Automation rules prohibit spam-like behavior, coordinated inauthentic actions, or metric inflation; violations lead to suspensions.[6]
- Best practices from tools:
  1. Use **aged, active, verified accounts** for higher safe limits.
  2. **Vary messages, add delays** (mimic human pacing).
  3. No links in initial DM; wait for reply.
  4. Tools auto-pause on rate limits.[3][4][6]
- Bans rare with compliance; X prompts human verification for sudden high volume even manually.[3]

```javascript
const { TwitterApi } = require('twitter-api-v2');

const client = new TwitterApi({
  appKey: 'YOUR_APP_KEY',
  appSecret: 'YOUR_APP_SECRET',
  accessToken: 'YOUR_ACCESS_TOKEN',
  accessSecret: 'YOUR_ACCESS_SECRET',
});

async function sendDM(conversationId, text) {
  try {
    const message = await client.v2.sendDm(conversationId, [{ text }]);
    console.log('DM sent:', message);
  } catch (error) {
    if (error.code === 429) {
      const retryAfter = error.headers['retry-after'];
      console.log(`Rate limited. Retry in ${retryAfter}s`);
      await new Promise(resolve => setTimeout(resolve, retryAfter * 1000));
    }
  }
}

sendDM('CONVERSATION_ID', 'Personalized message here');
```
